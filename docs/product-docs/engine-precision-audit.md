## 一、八字命理（bazi.js）— 🟡 80分

### ✅ 已修复（lunar-javascript精确计算）
- 四柱排盘：节气定月 ✅，日柱精确 ✅，时柱正确 ✅
- 纳音：精确 ✅
- 十二长生：精确 ✅
- 藏干：精确 ✅
- 十神：完整10×10映射 ✅
- 大运：含起运年龄，男女顺逆 ✅
- 流年：当前年份精确 ✅

### ⚠️ 存在的问题

#### P0 — 影响结果准确性

**1. 子时跨日分歧**
- 23:00-00:59 是子时，存在"早子时/晚子时"争议
- lunar-javascript 将 23:00 算作当日（早子时派）
- 部分命理师认为 23:00 后应算次日（晚子时派）
- 影响：23点出生的人日柱可能差一天
- 建议：提供"子时分界"设置选项（默认跟随lunar-js）

**2. 格局判断过度简化**
- 当前：直接取月干十神+"格"（如"劫财格"）
- 问题："劫财格"不是正统格局名，正统格局需看月令藏干透出
- 正统判法：月支藏干哪个透到天干，就取那个十神定格
- 如果都没透，看月支本气
- 影响：格局名称经常不准确，专业用户一眼看出问题
- 建议：实现正统格局判定逻辑（约50行代码）

**3. 身强身弱判断粗糙**
- 当前：日主五行在八字出现次数 >=3 身强，<=1 身弱
- 问题：
  - 没有考虑月令旺衰（月支对日主的生克关系权重最大）
  - 没有区分"生"和"助"的不同权重
  - 只统计地支本气，忽略藏干中气和余气
- 影响：身强弱是喜忌判断的基础，错了后面全错
- 建议：实现月令+得地+透出+合化的综合评分体系

#### P1 — 影响分析深度

**4. 五行统计只取本气**
- 当前：天干4个 + 地支本气4个 = 8个元素
- 问题：巳火藏丙火+庚金+戊土，只取丙火，低估金和土
- 建议：加权统计，本气1.0，中气0.5，余气0.3

**5. 喜用忌判断过于简化**
- 当前：身强则喜泄耗，身弱则喜生助
- 问题：没有考虑特殊格局（从格、化格等），也没有考虑调候用神
- 建议：先修复身强弱，喜忌自然更准

**6. 缺少流月/流日计算**
- ai-service层有简化的流月流日，但不是从lunar-javascript精确获取
- 建议：用 lunar-javascript 计算当前月和日的准确干支

## 二、星座运势（astrology.js）— 🔴 30分

### ❌ 严重问题

**1. 月亮星座完全虚假**
```javascript
// 当前代码：用hash伪随机生成
function getMoonSign(y,m,d){
  const idx=((y*13+m*29+d*7)%12+12)%12;
  return SIGNS[idx];
}
```
- 月亮每2.5天换一个星座，需要精确的月亮星历表
- 相邻两天（如8月15日和8月16日）结果毫无关联
- 用户如果知道自己的真实月亮星座，会立刻发现不对

**2. 上升星座完全虚假**
```javascript
// 当前代码：太阳星座 + hour/2 偏移
function getRising(m,d,h){
  const si = getSunSign(m,d).id index;
  return SIGNS[(si+Math.floor(h/2))%12];
}
```
- 上升星座需要：精确出生时间 + 出生地经纬度 + 恒星时计算
- 当前算法和真实结果完全无关

**3. 太阳星座边界有1天误差**
- 用固定日期分界（如水瓶座1月20日起），但实际分界每年因太阳精确位置不同而浮动1天
- 影响：星座分界日出生的人可能判断错误

### 修复方案
- 需要引入天文计算库（astronomia 或 Swiss Ephemeris WASM）
- 或者调用外部API（如 AstrologyAPI）获取精确行星位置
- 最低成本方案：从数据库查预计算的月亮星历表（1940-2030年每日月亮位置）

## 四、梅花易数（meihua.js）— 🟡 75分

### ⚠️ 问题

**1. 23点子时跨日**
- 23点属次日子时，但当前用 `new Date()` 的日期不变
- 起卦参数 `年+月+日` 应该用次日日期
- 影响：23:00-23:59 起卦结果和次日0:00-0:59不一致

**2. 时辰序号计算方式**
- 当前公式 `Math.floor(((h+1)%24)/2)+1` 对大部分小时正确
- 但传统梅花以"时辰"为单位（1-12），当前给出的是1-12没问题
- 已验证各小时输出符合预期

**3. 缺少互卦分析**
- 传统梅花易数除了本卦、变卦，还有互卦（由2-4爻和3-5爻组成）
- 互卦代表事情的过程和中间状态
- 影响：分析不够完整

### 修复建议
- 子时跨日：检测hour>=23时day+1
- 互卦：加10行代码即可

## 五、印度占星（vedic.js）— 🔴 15分

### ❌ 几乎全部虚假

| 数据 | 方法 | 准确度 | 影响 |
|------|------|--------|------|
| 太阳星座 | 简化公式 - 固定Ayanamsa | ±1星座 | 中 |
| 月亮星座 | hash伪随机 | 完全随机 | 致命 |
| 上升星座 | hour/2偏移 | 完全随机 | 致命 |
| Nakshatra | 基于假月亮 | 完全随机 | 致命 |
| Dasha大运 | 基于假Nakshatra | 完全错误 | 致命 |
| Ayanamsa | 固定24.17° | ±0.5° | 低 |

整个引擎除了太阳星座近似准确外，其余全部不可信。
Dasha大运是印度占星的核心——如果用户知道自己的真实Dasha，会立刻发现完全不对。

### 修复方案
- 必须引入 Swiss Ephemeris（WASM版或Node.js binding）
- 或者使用预计算的行星位置表
- 最低成本：月亮位置查表（和星座引擎共用）+ Ayanamsa动态计算
- Lagna仍需要完整的恒星时+地理位置计算
