| 4.7.2 | **机器人档案库** | 预生成50-100个机器人档案，不同五行/性格/年龄段/性别分布均匀 | 中 |

| 4.7.5 | **主动消息** | 机器人定时主动发消息（早安/晚安/今日运势分享/话题发起），模拟真人节奏 | 中 |

| 4.7.9 | **渐进退出** | 真实用户增长后，自动降低机器人比例：<100人→80%机器人，<1000→50%，<5000→20%，5000+→关闭 | 小 |

## 隐私保护体系（北美级 CCPA/CPRA/COPPA合规）

> 贯穿全开发阶段，以下为隐私保护的完整设计

### 数据分级

| 级别 | 数据 | 保护措施 |
|------|------|---------|
| S级（绝密） | 精确出生时间、完整八字四柱 | AES-256加密存储，客户端解密，不明文传输 |
| A级（高敏） | 日主/十神/格局、性格分析 | 加密存储，不对其他用户可见 |
| B级（中敏） | 匹配分数、AI占卜报告 | 本地缓存，服务端不持久化内容 |
| C级（低敏） | 昵称、星座、兴趣标签 | 正常存储，可对其他用户展示 |

### 对其他用户的信息隔离

```
默认可见：昵称、年龄段(25-30)、匹配等级(高/中/低)、兼容类型
需对方授权：精确年龄、星座、五行属性、匹配分数
永远不可见：出生日期、时辰、八字、AI报告、与他人匹配记录
```

### 3级隐私档位（用户自选）

```
标准模式（默认）：年龄段+匹配等级+兼容类型
开放模式：+ 精确年龄+星座+五行属性+匹配分数
保守模式：只显示昵称+大致距离
```

### 数据收集最小化

```
必须收集：出生日期（命理计算）、性别（匹配）、邮箱（验证）
可选收集：出生时辰、出生城市、昵称（需单独授权弹窗）
绝不收集：真实姓名、身份证号、精确GPS、通讯录、相册
```

### LLM调用隐私

```
发给AI：匿名出生数据（年月日时+性别）→ 无法关联到用户
不发给AI：真实姓名、手机号、位置、用户ID
AI结果：不持久化到服务端，只在客户端缓存
调用日志：只记录模式/耗时/token数，出生信息用****替代
```

### 用户权利（CCPA/CPRA）

```
知情权：查看所有已收集数据
可携带权：导出JSON/PDF
删除权：各项数据独立可删，注销72小时内完全清除
退出权：退出匿名分析/个性化推荐/第三方共享
更正权：随时修改出生信息
```

### 法律文件清单

| 文件 | 用途 | 阶段 |
|------|------|------|
| Privacy Policy（英+中） | 隐私政策，CCPA/CPRA合规 | Step 8 |
| Terms of Service | 用户协议 | Step 8 |
| DPA (Data Processing Agreement) | 数据处理协议 | Step 8 |
| CCPA Notice at Collection | 加州收集通知 | Step 8 |
| 命理分析免责声明 | 不构成专业建议 | Step 8 |
| Age Verification | COPPA合规（≥13岁） | Step 5 |

### App Store合规

```
Apple: Privacy Label（每项数据标注用途+是否追踪）+ ATT（不用IDFA）
Google: Data Safety Section（加密传输+可删除+共享清单）
敏感分类：命理属于Lifestyle，需声明不提供医疗/心理诊断
```
