### 4.7 LLM冷启动机器人系统（详细设计）

> 核心目标：项目冷启动期（0-5000真实用户），用AI机器人营造社区活跃氛围，让真实用户有人可匹配、有人可聊天、有内容可看。

| # | 子任务 | 说明 | 工作量 |
|---|--------|------|--------|
| 4.7.1 | **机器人人设引擎** | 基于八字生成完整虚拟人格：姓名/年龄/性格/爱好/说话风格/职业背景，每个机器人有独立prompt | 大 |
| 4.7.2 | **机器人档案库** | 预生成50-100个机器人档案，不同五行/性格/年龄段/性别分布均匀 | 中 |
| 4.7.3 | **自动匹配池** | 真实用户滑卡时，混入机器人卡片（真人优先，不足时补机器人），用户无法区分 | 中 |
| 4.7.4 | **1v1 AI聊天** | 匹配机器人后进入聊天，LLM基于该机器人人设+用户八字性格实时回复 | 大 |
| 4.7.5 | **主动消息** | 机器人定时主动发消息（早安/晚安/今日运势分享/话题发起），模拟真人节奏 | 中 |
| 4.7.6 | **社区内容生成** | 机器人自动发帖（命理感悟/今日心情/问题讨论），维持社区内容流 | 中 |
| 4.7.7 | **行为仿真** | 机器人模拟真人行为模式：非即时回复(延迟30s-5min)、打字动画、在线/离线状态切换、阅读已读 | 中 |
| 4.7.8 | **真人接管** | 后台可随时将机器人对话转为人工客服接管，用户无感知 | 中 |
| 4.7.9 | **渐进退出** | 真实用户增长后，自动降低机器人比例：<100人→80%机器人，<1000→50%，<5000→20%，5000+→关闭 | 小 |
| 4.7.10 | **标记系统** | 内部标记机器人账号（用户不可见），运营后台可查看机器人/真人比例 | 小 |
